# Penmanship Practice — 설계 문서

작성일: 2026-07-03
상태: 초안 (사용자 리뷰 대기)

## 한 줄 요약

Obsidian 노트 안에서 마우스로 글씨(알파벳·숫자·한글 등)를 따라 쓰며, 목표 글자와 얼마나 겹치는지 채점하거나 자유롭게 연습하고, 그 결과를 이미지로 노트에 남기는 플러그인.

## 목적

두 가지 훈련을 동시에 지원한다.

1. **손글씨 미적 훈련** — 목표 글자 모양에 가깝게 예쁘게 쓰기.
2. **마우스 컨트롤 훈련** — 그림 그리기용 손 훈련. 외곽선 안에 정확하게 마우스로 긋기.

두 목적 모두 "목표 글자와 사용자 드로잉의 겹침 정도"라는 단일 지표로 자연스럽게 커버된다.

## 사용자가 확정한 요구사항

- 채점 모드(겹침 %)와 자유 연습 모드 **둘 다** 제공.
- Excalidraw처럼 **노트 안에서** 글씨를 쓴다.
- 리본(사이드바)에 아이콘.
- 연습 결과를 **그린 이미지로 노트에 남긴다**.

## 기본값으로 결정된 사항 (사용자 부재 시 판단 — 리뷰에서 변경 가능)

- **연습 대상**: 폰트 기반 렌더링이라 알파벳·숫자·한글·한자를 문자별 데이터 없이 모두 지원. MVP 확인은 알파벳/숫자/한글로.
- **캔버스 방식**: 노트 안 코드블록 위젯(` ```penmanship `). Excalidraw식 별도 파일타입보다 구현이 가볍고, 결과 로그가 노트 흐름에 자연스럽게 쌓인다.
- **채점 방식**: 픽셀 겹침(IoU 계열). 아래 "채점 방식 결정" 참조.

## 채점 방식 결정

세 후보 중 **픽셀 겹침**을 채택.

| 방법 | 방식 | 채택 여부 |
|---|---|---|
| 픽셀 겹침 (IoU) | 목표 글자를 굵은 형태로 래스터화하고 사용자 획도 래스터화해 겹치는 픽셀 비율 계산 | **채택** |
| 획순 매칭 | 문자별 획순 데이터와 순서 비교 | 미채택 (문자별 데이터 필요, 한글 방대) |
| DTW 경로 비교 | 목표 경로와 사용자 경로를 시계열로 비교 | 미채택 (문자별 경로 데이터 필요) |

채택 이유: (1) 문자별 데이터가 필요 없어 폰트만 있으면 어떤 문자든 동작 → 한글·한자 확장이 공짜. (2) 미적 훈련(모양 유사도)과 마우스 컨트롤 훈련(외곽선 안에 긋기)을 하나의 지표로 동시 충족. (3) 구현 단순.

트레이드오프: 획순은 채점하지 못한다(모양만 본다). MVP 목적상 허용.

## 아키텍처

세 개의 독립 유닛으로 분리한다. 각각 하나의 책임만 가지며 인터페이스로 소통한다.

### 1. PracticeBlock — 코드블록 렌더러
- 책임: 노트의 ` ```penmanship ` 코드블록을 파싱하고 연습 위젯 DOM을 구성.
- 입력: 코드블록 본문(YAML — `target`, `mode`, `guide` 등).
- 의존: DrawingCanvas, Scorer, Obsidian API(파일 저장·노트 편집).
- 코드블록 본문 예:
  ```
  target: 한
  mode: scored        # scored | free
  guide: outline      # outline | filled | none
  size: 300
  ```

### 2. DrawingCanvas — 그리기 엔진
- 책임: HTML5 Canvas에 마우스로 획을 그리고, 목표 글자를 반투명 가이드로 배경 표시. 그리기 로직만 담당하며 채점을 알지 못한다.
- 입력: 목표 글자, 가이드 스타일, 크기.
- 출력: 사용자가 그린 획의 래스터(ImageData) 또는 PNG blob, 지우기/되돌리기.
- 의존: Canvas API만. Obsidian·채점과 독립.

### 3. Scorer — 채점기
- 책임: 목표 글자 마스크와 사용자 드로잉 마스크를 받아 겹침 %를 계산.
- 인터페이스: `score(targetMask: ImageData, userMask: ImageData): number` (0–100).
- 순수 함수. 캔버스·노트와 완전 독립 → 단위 테스트 가능.
- 지표(기본값 확정): `precision`(사용자 획 중 목표 안에 든 비율)과 `recall`(목표 영역 중 사용자가 덮은 비율)의 조화평균(F1)을 0–100으로 환산. 즉 "목표를 잘 덮으면서 밖으로 안 삐져나갈수록" 고득점. 목표 글자는 채점 시 약간 두껍게(허용 오차) 래스터화해 마우스의 미세한 흔들림을 관대하게 처리.

## 데이터 흐름

```
노트의 penmanship 블록
  → PracticeBlock 파싱 (target="한", mode="scored")
  → DrawingCanvas 렌더 (배경에 "한" 가이드 표시)
  → 사용자가 마우스로 획을 그림
  → [채점 모드] "채점/저장" 클릭
       → Scorer가 목표 마스크 vs 사용자 마스크 겹침 % 계산 → 화면 표시
       → 캔버스를 PNG로 vault 첨부 폴더에 저장
       → 노트 블록 아래에 ![[결과.png]] + "정확도 82% · 2026-07-03" 삽입
  → [자유 모드] 채점 없이 그림. 원하면 이미지만 저장.
```

## UI

- **리본 아이콘**: 클릭 시 작은 모달로 목표 글자를 입력받아, 현재 커서 위치에 `penmanship` 코드블록 템플릿을 삽입.
- **연습 위젯**(코드블록 자리): 캔버스 + 목표 글자 가이드 + 버튼(지우기, 되돌리기, 채점/저장, 가이드 토글).

## 모드

- **채점 모드(scored)**: 가이드 표시 + 저장 시 겹침 % 채점 + 결과(이미지·점수·날짜) 노트 삽입.
- **자유 모드(free)**: 가이드 on/off 가능, 채점 없음, 원하면 이미지만 저장.

## 기록 방식

그린 캔버스를 PNG로 vault에 저장하고, 노트에 `![[이미지]]` 임베드 + (채점 모드) 점수·날짜 한 줄 추가.

## 에러 처리

- 목표 글자 미지정 → 위젯에 안내 메시지 표시.
- 캔버스 저장 실패(권한 등) → Notice로 알림.
- 빈 캔버스 채점 시도 → "먼저 그려주세요" (0% 처리하지 않음).

## 테스트

- **Scorer**: 순수 함수라 단위 테스트. 완전 겹침 = 100%, 안 겹침 = 0%, 절반 겹침 ≈ 50%, 목표 밖 초과 시 감점 확인.
- **DrawingCanvas / PracticeBlock**: 실제 Obsidian vault에서 수동 검증(그리기·저장·삽입 동작).

## 기술 스택

- TypeScript + Obsidian Plugin API.
- esbuild 표준 빌드.
- 외부 라이브러리 없음(Canvas API만 사용).

## MVP 범위 (YAGNI)

포함:
- 알파벳·숫자·한글 (폰트 기반 자동 지원, 획순 데이터 불필요)
- 픽셀 겹침 채점 1종
- 채점 / 자유 모드
- PNG 저장 + 노트 삽입
- 리본 아이콘 → 블록 삽입

제외(나중):
- 획순 애니메이션 안내, 획순 채점
- 연습 통계 대시보드, 리더보드
- 필압(마우스에는 없음)
