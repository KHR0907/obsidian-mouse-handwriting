# Penmanship Practice — 설계 문서 (개정 2)

작성일: 2026-07-03
상태: 재설계 (사용자 피드백 반영)

## 개정 이유

초안은 "노트 안 코드블록 위젯"으로 접근했으나 방향이 틀렸다. 올바른 방향:

> **Excalidraw처럼 전용 페이지(새 파일타입)를 통째로 쓰는, 하나의 내부 서비스.**
> **한컴타자연습의 손글씨 버전 = 마우스 "따라쓰기 연습장".**
> 긴 글 연습·단어 연습처럼 **문제 세트를 내주고 따라 쓰게** 한다.

## 한 줄 요약

`.penmanship` 파일을 열면 전용 연습 뷰가 뜬다. 내장 문제 세트(자모·단어·긴 글)를 골라, 칸마다 목표를 보여주고 마우스로 따라 쓰게 하며, 칸별·세션별로 겹침 정확도를 채점하는 손글씨 따라쓰기 연습장.

## 목적

- 손글씨 미적 훈련 + 마우스 컨트롤 훈련.
- 한컴타자연습처럼 **문제를 주고 순서대로 따라 쓰며** 진행/채점하는 연습 서비스.

## 핵심 개념

- **연습장(.penmanship 파일)** = 하나의 연습 세션. 선택한 문제 세트 + 진행 상태 + 칸별 점수를 담는다.
- **문제 세트(ProblemSet)** = 따라 쓸 목표들의 묶음. 유형별로 내장 제공.
- **칸(cell)** = 문제 하나(글자/단어/어절). 각 칸은 목표 가이드 + 그리기 캔버스 + 점수.

## 내장 문제 세트 (다양하게)

| 유형 | 예시 | 칸 단위 |
|---|---|---|
| 한글 자모 | ㄱㄴㄷㄹ… / ㅏㅑㅓㅕ… | 자모 1개 |
| 알파벳 | A–Z, a–z | 글자 1개 |
| 숫자 | 0–9 | 글자 1개 |
| 단어 연습(한글) | 사과, 나무, 하늘, 바다… | 단어 1개 |
| 단어 연습(영어) | cat, dog, sun… | 단어 1개 |
| 긴 글 연습 | 문장/문단을 어절 단위로 분할 | 어절 1개 |

문제 세트는 순수 데이터 모듈로 정의하며, 사용자가 나중에 커스텀 세트를 추가할 수 있는 구조로 둔다(초기엔 내장만).

## 아키텍처

### 1. PenmanshipView (ItemView + 파일타입 등록)
- `.penmanship` 확장자를 등록하고, 파일을 열면 뜨는 전용 뷰.
- 상단: 문제 세트 선택 / 진행바(3 / 20) / 전체 정확도.
- 본문: 현재 칸의 목표 표시 + DrawingCanvas + 버튼(지우기·되돌리기·다음).
- 파일 로드/세이브(진행 상태 JSON) 담당.

### 2. ProblemSets (순수 데이터)
- 내장 세트 목록과 각 세트의 문제 배열을 제공.
- 긴 글은 문자열을 어절 단위로 분할하는 헬퍼 포함.

### 3. PracticeSession (순수 로직, 테스트 가능)
- 상태: 현재 세트, 현재 칸 인덱스, 칸별 점수 배열.
- 동작: `recordScore(i, score)`, `next()`, `prev()`, `overall()`(평균 정확도), `isComplete()`.
- 뷰·캔버스와 독립 → 단위 테스트.

### 4. DrawingCanvas (기존 재사용)
- 칸 하나의 그리기 + 목표 가이드. 채점용 마스크 추출.
- 기존 구현 그대로 사용(문자 하나든 단어든 폰트로 렌더).

### 5. Scorer (기존 재사용)
- 목표 마스크 vs 사용자 마스크 겹침 F1 → 0–100. 기존 구현 그대로.

## 파일 포맷 (.penmanship)

JSON:
```json
{
  "version": 1,
  "setId": "korean-words-basic",
  "currentIndex": 3,
  "scores": [88, 91, 0, null, ...]
}
```
- 이미지 저장은 옵션(세션 완료 시 "결과 이미지 내보내기" 정도). MVP에선 점수 기록 중심.

## 데이터 흐름

```
.penmanship 파일 열기
  → PenmanshipView가 JSON 로드 → PracticeSession 복원
  → 현재 칸의 목표(예: "사과")를 DrawingCanvas 가이드로 표시
  → 사용자가 마우스로 따라 씀 → "채점 & 다음"
      → Scorer로 겹침 % 계산 → session.recordScore()
      → session.next() → 다음 칸 목표 표시
  → 마지막 칸이면 세션 완료 화면(전체 평균 정확도, 다시하기)
  → 상태를 .penmanship 파일에 저장
```

## UI

- **리본 아이콘 / 명령어**: "새 따라쓰기 연습장" → 세트 선택 → 새 `.penmanship` 파일 생성 후 뷰 열기.
- **뷰 상단바**: 세트명, 진행(현재/전체), 전체 정확도, 세트 바꾸기.
- **뷰 본문**: 큰 목표 텍스트/가이드가 깔린 캔버스, 아래 버튼(지우기·되돌리기·이전·채점&다음·가이드 토글).
- **완료 화면**: 평균 정확도, 칸별 점수 요약, 다시하기/새 세트.

## 모드

- **채점 모드**: 각 칸 겹침 % 채점 + 세션 평균.
- **자유 모드**: 채점 없이 세트를 따라 쓰기만(가이드 on/off). 세트/칸 진행은 동일.

## 에러 처리

- 잘못된/빈 `.penmanship` 파일 → 새 세션으로 초기화.
- 빈 캔버스로 채점 시도 → "먼저 써주세요".
- 이미지 내보내기 실패 → Notice.

## 테스트

- **PracticeSession**: 순수 로직 단위 테스트(진행/점수/평균/완료 판정).
- **Scorer**: 기존 단위 테스트 유지.
- **PenmanshipView / 파일타입**: 실제 Obsidian에서 수동 검증.

## 기술 스택

- TypeScript + Obsidian Plugin API(`registerView`, `registerExtensions`).
- esbuild 빌드. 외부 라이브러리 없음(Canvas API만).

## MVP 범위 (YAGNI)

포함:
- `.penmanship` 파일타입 + 전용 뷰
- 내장 문제 세트(자모/알파벳/숫자/한글단어/영어단어/긴글 최소 1개씩)
- 칸 단위 따라쓰기 + 채점(F1) + 세션 진행/평균
- 채점 / 자유 모드
- 리본 → 새 연습장 생성

제외(나중):
- 사용자 커스텀 문제 세트 편집 UI
- 획순 채점/애니메이션
- 통계 대시보드, 결과 이미지 갤러리
